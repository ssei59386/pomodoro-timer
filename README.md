# 定期テスト学習進捗管理アプリ（Phase 0 最小版）

中高生が定期テストで点数を最大化するための、学習進捗管理スマホアプリ。
「理解度を測る → 点数が最大になるよう章を割り当てる → また測って計画を組み替える」
という基本ループを回します。

このリポジトリは設計仕様書の **Phase 0（最小版）** を実装したものです。

## できること（Phase 0）

- 対象教科は **数学・理科** の2教科
- 章ごとの理解度を **自己申告（初回）＋ 演習正答率と自己申告（以降）** で管理
- **曜日ごとの空き時間帯**を登録し、その日の合計分数に応じて勉強計画を立てる（手動入力。将来カレンダー連携への差し替えを想定した設計）
- **優先度スコア**による「次にやるべき章」の決定（貪欲法・1章集中）
- セッション記録（章・時間・正答率・手応え）の入力と、それによる理解度の更新
- 「今日やること」の提示
- 章ごとの理解度ダッシュボード（現在 vs 目標、テストまでの残り日数）
- データはすべて **端末ローカル（localStorage）に保存**。バックエンド・ログイン不要

> 忘却曲線による減衰・AI・5教科対応・丸暗記モードなどは Phase 1 以降で、本版には含みません。

## 技術スタック

- React + TypeScript
- Vite（モバイル向け PWA・ホーム画面に追加可能）
- 状態管理は React Context、保存は localStorage

## 画面構成

1. 初期設定 / オンボーディング（教科・テスト日・章・配点・自己申告・曜日ごとの空き時間帯）
2. ホーム（今日やること）
3. セッション記録
4. 理解度ダッシュボード
5. 設定（編集・データリセット）

## コアロジック（純粋関数 `src/logic.ts`）

- 理解度の更新：`observed = 0.7×正答率 + 0.3×(自己申告/5)`、平滑化 `α=0.5`
- 優先度スコア：`priority = pointWeight × max(目標−理解度, 0) × proximity`
  （`proximity = 1 / 残り日数`）
- 計画生成：priority の高い順に、1日の勉強可能時間を1章ずつ集中して割り当てる

## セットアップ

```bash
npm install      # 依存関係をインストール
npm run dev      # 開発サーバを起動（http://localhost:5173）
npm run build    # 本番ビルド（dist/ に出力）
npm run preview  # ビルド結果をローカルで確認
npm test         # コアロジックのユニットテスト（vitest）
```

スマホで使うときは開発/プレビューの URL をブラウザで開き、「ホーム画面に追加」してください。

## データの取り扱い

データはこの端末（ブラウザの localStorage）内にのみ保存され、サーバー送信・ログインはありません。
設定画面からいつでもリセットできます。
